# 开发日志

> 本文件记录项目的整体改进过程与**每次开发的思路/决策**,供日后回顾,也方便其他智能体(或新协作者)快速了解项目脉络。

> `日期` · **做了什么** · **思路/决策(为什么这么做)** · **结果(PR/commit、验证)**。
> 偏好:本地实现 → `npm run build` → 用户实测 → 确认后再 push/PR;合并保留独立 commit;版本暂留 0.1.0。

## 项目一句话
把 B 站动态按"关注分组"分 Tab 浏览的浏览器扩展(WXT MV3 + Preact + Dexie + Zod),
解决"关注过多时低频优质 UP 被混合时间流淹没"的问题,并在后台主动补全低频 UP 的动态。数据全本地(IndexedDB)。
架构与目录见 [README](README.md)。

## 开发记录(最新在上)

### 2026-06-29 · 同步抗标签页休眠 + 倒计时改读后台真实时间(本地待测)

**做了什么**
- 明确**后台 alarm 是周期同步的唯一来源**(标签页冻结/丢弃都不影响);仪表盘不再自己驱动同步,只负责"显示 + 追上"。
- 倒计时改读后台 alarm 的真实 `scheduledTime`(`getUiState` 暴露 `meta.next_sync_at`);仪表盘加 60s 轮询 + 唤醒刷新 + 倒计时归零时立即刷新(10s 节流)以追上重排时间;打开仪表盘若数据已过期则补一次同步。

**思路/决策**
- 浏览器为省内存会冻结(暂停 JS 计时器)或丢弃(重载)标签页,所以"靠标签页驱动定时同步"本就不可靠;周期任务本应归后台 SW alarm(MV3 的设计意图),仪表盘退化为"诚实显示 + 唤醒追上"。
- 经对抗式审查修两处**倒计时显示**问题(非同步本身):首开时 alarm 可能尚未创建 → `getUiState` 兜底创建;归零后最多 10s 内自纠重排时间。审查另否决了两条误报(改间隔双同步、设置加载前用默认间隔)。

**结果**:`typecheck` / 13 单测 / `build` 全绿;待本地实测(让标签页休眠后再回来,看倒计时是否追上、数据是否已被后台同步)。

### 2026-06-29 · 浮动按钮:限定动态页 + 可拖动(本地待测)

**做了什么**
- 内容脚本 `matches` 收窄为只在动态页 `https://t.bilibili.com/*` 注入(原来还在首页/播放页注入,会挡到其他按钮)。
- 浮动按钮改为可拖动(pointer 事件):拖动阈值 5px 区分点击与拖动;位置存 `chrome.storage.local`,刷新后保留;窗口缩放时夹回视口内。

**思路/决策**
- 收窄 `matches` 比"全站注入 + 条件隐藏"更干净——其他页面根本不注入。`host_permissions` 仍保持全站(后台 API 调用需要),与内容脚本注入范围互不影响。
- 拖动结束会附带触发一次 click,用元素 `dataset` 标志吞掉它,并在每次 `pointerdown` 重置标志防卡死;定位从 CSS 的 right/bottom 锚点切到显式 left/top。

**结果**:`typecheck` / 13 单测 / `build` 全绿,manifest 的 content_scripts 仅 `t.bilibili.com`;待本地实测(动态页可见、首页/播放页不再出现、可拖动且刷新后位置保留)。

### 2026-06-28 ~ 06-29 · 首轮深度改进(PR #1、#2,均已并入 main)

**整体思路**
- 先并行通读各子系统建立全局认识,再做"对抗式 Bug 审查"(多 agent 查找 + 独立复核),只采纳复核确认的问题,避免误报。
- 凡涉及运行时行为的改动,都先本地构建让用户实测,实测发现的真问题(如 -352、`[动态]`、刷新不更新)才是验证主力——typecheck/单测只能兜底。

**PR #1 — 修复 + 基础改进**
- 9 个复核确认的 Bug:空间扫描游标改为按 mid 推进(防漏扫/重复)、`updateCount24h` 写入事务化(防 M1 同步与空间补数竞态互相覆盖)、Port 监听器清理、切 Tab 乱序响应用 `active` 标志丢弃、四象限中位数边界 `>=`→`>`、AI 响应 `JSON.parse` 兜底、WBI/collectPics 边界。
- Tab 条支持鼠标滚轮横滚(修无触控板不可用);数据保留策略(dynamics 60 天 + 每 UP 留 10 条,viewLogs 35 天,纯函数 + 单测);Windows 部署文档(`engines` 字段 + README 写清"先 build 再加载 `.output/chrome-mv3`")。
- **-352 风控**:空间接口 `getSpaceDynamics` 补 WBI 签名 + 网页端指纹参数(关键洞察:`feed/all` 宽松、`feed/space` 严)。
- **`[动态]` 解析**:`summarizeDynamic` 覆盖 common / 番剧 / 音乐 / 课程 等卡片类型。

**PR #2 — 三个新功能**
- **运行日志**:每小时心跳快照 + 同步/补数/清理事件记录 + 主面板「导出日志」一键复制(配合文字描述"双重锁定"问题);自清理 14 天 / 5000 条;经一轮审查加固(防 unhandled rejection、`pruneLogs` 走事务、剪贴板权限处理)。
- **同步倒计时**:去掉分组里误导的"刷新"(它只重读本地缓存、拉不到新内容,且时间不更新)。思路:能真正拉到新内容的只有"同步",所以倒计时应是**全局同步**而非分组级刷新——概念要诚实。顶部改为「上次同步 X · 距下次同步 mm:ss」,到点自动同步、可手动同步,间隔取自 `syncIntervalMinutes`。直播页保留 30s 自刷(真联网拉取)。
- **设置移入面板**:工具栏「设置」按钮 + 共享 `SettingsForm`(面板与选项页同源),不必再进浏览器深处的选项页。

**结果**:`typecheck` / 13 单测 / `build` 全绿 + Mac & Windows 实测通过;两个 PR 均以 `--merge`(保留独立 commit)并入 `main`,版本保持 0.1.0。

**已知待办 / 风险**
- 四象限分析 UI 未做(数据层已完成)。
- 核心解析(`parseDynamicItem`)与同步流程缺测试,恰是最易因 B 站接口变动而静默出错处——建议补"真实 raw JSON 喂解析器"的快照测试。
